    A simpler game AI that never loses using complete game tree analysis.

### Idea analysis 

1. Medical Diagnosis Expert System 

    a. The system would collect patient information and data to best determine the patient’s diagnosis. There would be a series of inputs for symptom collection and additional inputs produced for certain symptoms. You could perhaps factor in patient history and family history as well. The system should suggest a diagnosis based on information given and give the user the chance to edit or add to the symptoms given until a good diagnosis is reached. 

    b. This could work a lot like how the recipe recommender worked in the example given in class. Collect a comma-separated list of symptoms to compare a list of symptoms for each ailment. Collect benchmark values in a separate list, like blood pressure and temperature. 

2. Smart Home Automation Controller 

    a. This system would be more automated, requiring less input than the medical diagnosis system. Based on the time and day, you could determine the room temperature, appliances running, blind level, watering plants, feeding animals, and other functions. 

    b. For development and testing, it may be better to enter a day and time. Each function should have a type, normal, toggle, or custom, and a list of times. Functions that are normal will do a task at the time specified, like feed animals. Functions that toggle will turn on or off at the time specified, like blinds up or down. Functions that are custom will need extra programming, like controlling room temperature where a time and a temperature is needed to complete each task. 

3. Tic-Tac-Toe 

    a. The system would have to keep track of a board and positioning of pieces, as well as loop to gather input on the user’s moves. After each user move, the system must determine how to place its own pieces. This means that it needs to determine where to place the next piece to increase its chances of winning. 

    b. Start with a new board and a prompt and input for the user’s move. The board should be labeled with top, middle, bottom, left, and right. Once a valid move is played, the board is updated. An opposing move should be determined by send the board through a series of tests and returning a list of possible moves. The system then chooses a move, updates the board, and asks for user input again.

```
def get_ai_move(board, ai_symbol, opponent_symbol): 

    Priority 1: Check for winning move 
    move = find_winning_move(board, ai_symbol) 
    IF move exists THEN RETURN move 

    Priority 2: Block opponent's winning move 
    move = find_winning_move(board, opponent_symbol) 
    IF move exists THEN RETURN move 
    
    # Priority 3: Create a fork 
    move = find_fork_opportunity(board, ai_symbol) 
    IF move exists THEN RETURN move 
    
    # Priority 4: Block opponent's fork 
    move = block_opponent_fork(board, ai_symbol, opponent_symbol) 
    IF move exists THEN RETURN move 
    
    # Priority 5: Take center 
    IF board[4] is empty THEN RETURN 4 
    
    # Priority 6: Take opposite corner 
    corners = {0: 8, 8: 0, 2: 6, 6: 2} 
    FOR each corner, opposite IN corners: 
    IF board[corner] == opponent_symbol AND board[opposite] is empty 
    THEN RETURN opposite 
    
    # Priority 7: Take any empty corner 
    FOR corner IN [0, 2, 6, 8]: 
    IF board[corner] is empty THEN RETURN corner 
    
    # Priority 8: Take any empty side 
    FOR side IN [1, 3, 5, 7]: 
    IF board[side] is empty THEN RETURN side 
```

### Helper Functions 
```
# find_winning_move(board, symbol) 
FOR each winning pattern [a, b, c]: 

    IF board[a] == symbol AND board[b] == symbol AND board[c] is empty 
    THEN RETURN c 

    IF board[a] == symbol AND board[c] == symbol AND board[b] is empty 
    THEN RETURN b

    IF board[b] == symbol AND board[c] == symbol AND board[a] is empty 
    THEN RETURN a 

RETURN None 
```

```
# find_fork_opportunity(board, symbol) 
FOR each empty position: 

    Place symbol temporarily 
    Count how many winning patterns now have exactly 2 of our symbols 
    IF count >= 2 THEN this is a fork 
    Remove temporary symbol 

RETURN fork position if found, else None
```

## Part 4 - Reflection

### How does your rule-based system work?
The Tic Tac Toe class sets up base functions/methods to interact with the board by verifying valid moves, making moves, and determining game state. The AI helpers search the board for a certain condition or state that the AI can take advantage of or defend against. The main AI function focuses on priority of these functions, which should come first vs last. The main gameplay loop sets up the game, collects input, determines turn, and outputs the results of each move. There are three main pieces that work together with some helper functions. Those main pieces are the game containing the board, the AI player logic, and the main gameplay loop containing player UI/UX.
